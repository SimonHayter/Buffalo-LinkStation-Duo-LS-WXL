[] Web Server Settings
Click on [Modify Settings] to change settings.

[] Web Server
Enable to use the Web server.

[] Port Number
Choose the port for web server traffic. The default port is 81.

[] Web Server Publishing Folder
Enable to use the Web server. Three directories will be created in the Web Server Publishing Folder, htdocs, cgi-bin, and log:

(web server publishing folder, ie. web)/htdocs/
This folder is used to store the contents of your webserver and the HTML files or PHP scripts you would like to include. Example: When you set up a share folder called “web”, the path via SMB will be \\LinkStation_name\web\htdocs.

(web server publishing folder, ie. web)/cgi-bin/
Use this folder to store your CGI (Common Gateway Interface) scripts. Perl is the only supported language. Script extensions should be either .pl or .cgi.

(web server publishing folder, ie. web)/log/
Logs from the web server are stored in this folder.

[] Testing Web Server Functionality
Navigate to http://LinkStation_IP_address:port_number. For example, http://192.168.11.150:81. If the web server is configured correctly, phpinfo will be visible. Phpinfo shows the PHP interpreter settings.

[] php.ini
You can change PHP language interpreter settings (advanced users only). php.ini may be imported or edited from a connected PC.

Note: PHP scripts may not work as expected after you edit php.ini. You may go back to the default php.ini by clicking on [Restore Default Settings].
